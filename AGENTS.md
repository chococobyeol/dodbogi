# Dodbogi 작업 지침

## 기본 원칙

- 구현할 때 임의로 추측해서 만들지 않는다. 비슷한 기능, 동작, UX 구현 예시를 먼저 찾아보고 참고한 뒤 구현한다.
- UI/UX는 혼자 감으로 배치하지 않는다. 유사 앱, 공개 예시, 기존 참조 구현을 확인한 뒤 현재 앱 스타일에 맞게 정리한다.
- Magpie parity 관련 기능은 로컬 참조 코드 `D:\Projects\dodbogi\.codex\external\Magpie`를 먼저 확인한다.
- 기존 기능을 망가뜨리면서 새 기능이나 UI를 추가하면 실패다. 새 기능보다 기존 기능 정상 동작이 우선이다.
- 검증은 smoke test만으로 끝내지 않는다. 사용자가 실제 실행하는 환경과 최대한 비슷하게 검증한다.
- 검증 결과를 보고할 때는 어떤 exe를 실행했는지, 어떤 기능을 직접 확인했는지, 어떤 항목이 실패/미확인인지 명확히 쓴다.

## 작업 전 필수 확인

- 수정 전에 현재 앱의 구현된 기능 목록을 먼저 확인한다.
- 작업 범위가 UI뿐이어도 설정 저장, 단축키, 런타임, 트레이, 종료, 로그, 스크린샷 같은 기존 기능과 연결될 수 있으면 회귀 가능성으로 본다.
- 기존 기능을 직접 수정하지 않는 경우에도, 결과적으로 기존 기능이 깨질 수 있으므로 완료 전에 전체 핵심 기능을 재검증한다.
- 런타임/핫키/설정 파일/프로파일/트레이/마우스 입력/창 스타일 코드는 서로 영향을 주므로, 그 주변을 수정했다면 반드시 실제 실행 검증을 한다.

## Windows 실제 실행 검증 절차

Windows 앱 변경 후에는 아래 절차를 생략하지 않는다.

1. 기존 `dodbogi.exe` 프로세스를 모두 종료한다.
2. `cargo build -p dodbogi-app --release`로 빌드한다.
3. `target\release\dodbogi.exe`를 루트 `D:\Projects\dodbogi\dodbogi.exe`에 복사한다.
4. 반드시 루트 `D:\Projects\dodbogi\dodbogi.exe`를 직접 실행한다. 다른 경로의 exe로 검증하지 않는다.
5. 실제 실행 화면을 캡처하거나 직접 확인해서 UI 겹침, 잘림, 잔상, 정렬, 아이콘 의미, 사용 불편 요소를 눈으로 판단한다.
6. smoke test, unit test, build 성공은 보조 증거일 뿐이다. 실제 기능 동작 확인을 대체하지 않는다.

## 전체 기능 회귀 검증 체크리스트

완료 보고 전, 현재 구현된 기능을 가능한 한 전부 확인한다. 최소 기준은 아래와 같다.

### 실행/종료/창 상태

- 루트 `dodbogi.exe`가 정상 실행된다.
- X 버튼으로 종료하면 실제로 종료되고 프로세스가 남지 않는다.
- 트레이로 최소화가 정상 동작한다.
- 트레이 아이콘 클릭 또는 메뉴로 다시 복구된다.
- 앱 창 제목, 앱 아이콘, 작업표시줄/Alt-Tab 동작이 이상하지 않다.
- 앱 창 크기 변경 시 잔상, 검은 영역, 컨트롤 겹침이 없다.

### 프로파일 UI

- 기본 프로파일이 표시된다.
- 새 프로파일 추가가 동작한다.
- 프로파일 삭제가 동작한다.
- 프로파일 이름 변경이 동작한다.
- 이름 편집 중 클릭, Enter, Escape, 포커스 이동 동작이 자연스럽다.
- 긴 프로파일 이름이 칸 밖으로 넘치지 않는다.
- 프로파일 선택 변경이 설정 값과 화면에 일관되게 반영된다.

### 글로벌 설정

- 설정 창이 정상 표시/닫힘된다.
- 언어 변경이 실제 UI에 반영되고 저장된다.
- 기본값 초기화가 동작하고 UI 위치/정렬이 깨지지 않는다.
- 로그 출력 체크박스가 저장되고, 켜면 로그 창/출력이 실제로 표시된다.

### 단축키 설정

- 각 단축키 변경 버튼이 올바른 대상 단축키를 연다.
- 변경 전 단축키와 변경 후 단축키가 겹쳐 보이지 않는다.
- Ctrl/Alt/Shift/Win/단일 키 조합이 강제로 제한되지 않고 정상 입력된다.
- 적용 버튼은 실제 설정 저장과 런타임 핫키 재등록까지 반영한다.
- 취소 버튼은 변경을 버린다.
- 단축키 충돌이나 빈 단축키 처리 결과가 이상하지 않다.

### 창 확대

- 테스트용 일반 창을 띄운 뒤 창 확대 단축키가 실제로 동작한다.
- 확대 오버레이가 보이고 배율이 설정값대로 적용된다.
- 확대 중 원본 창 클릭, 호버, 드래그, 제목표시줄 이동, 모서리 크기 조절이 가능하다.
- 확대 중 커서가 사라지거나 두 개 보이거나 순간이동하지 않는다.
- 확대 중 다른 앱을 위로 올릴 수 있고, 포커스/항상위 동작이 사용을 방해하지 않는다.
- 확대 해제 후 마우스 속도, ClipCursor, 포커스, 창 상태가 원래대로 복구된다.

### 부분 확대

- 부분 확대 단축키가 실제로 동작한다.
- 마우스 주변 원본 영역이 설정된 범위와 배율로 확대된다.
- 창 확대가 이미 켜진 상태에서도 부분 확대가 의도대로 동작한다.
- 부분 확대 해제 후 커서/포커스/창 상태가 정상 복구된다.

### 확대 설정

- 창 확대 배율 입력이 저장되고 실제 창 확대에 반영된다.
- 부분 확대 배율 입력이 저장되고 실제 부분 확대에 반영된다.
- 부분 확대 영역 크기 입력이 저장되고 실제 부분 확대 범위에 반영된다.
- 숫자 입력 중간 상태는 허용하되, 최종 적용 시 하한/상한이 정상 처리된다.
- 잘못된 입력이 앱 종료나 설정 파일 손상을 일으키지 않는다.

### 스크린샷

- 창 확대 스크린샷 단축키가 동작한다.
- 부분 확대 스크린샷 단축키가 동작한다.
- 각각의 저장 경로 입력이 저장된다.
- 찾아보기 버튼이 폴더 선택 후 해당 경로를 반영한다.
- 저장 경로가 비어 있으면 루트 exe 폴더에 저장된다.
- 저장 파일이 실제로 생성되고, 파일이 열 수 있는 정상 이미지/출력물이다.

### 로그/설정 파일

- 설정 파일 저장 후 앱 재실행 시 값이 유지된다.
- 설정 마이그레이션이 기존 사용자 설정을 불필요하게 덮어쓰지 않는다.
- 로그 파일 또는 로그 창에 핵심 이벤트가 기록된다.
- 앱 종료 후에도 임시 상태나 마우스 상태가 남지 않는다.

## 완료 금지 조건

아래 중 하나라도 해당하면 완료라고 말하지 않는다.

- 루트 exe를 직접 실행하지 않았다.
- 구현된 전체 기능 회귀 체크리스트를 확인하지 않았다.
- 특정 기능을 못 확인했는데 완료처럼 보고했다.
- smoke/unit test만 통과하고 실제 동작을 확인하지 않았다.
- UI 화면을 직접 보지 않고 UI를 완료 처리했다.
- 창 확대, 부분 확대, 트레이 복구, 종료, 단축키, 설정 저장 중 하나라도 깨진 상태다.
- 테스트 실패, 프로세스 잔류, 마우스 속도 미복구, 커서 이상, 확대창 비정상 표시가 남아 있다.

## 보고 방식

완료 보고에는 반드시 아래를 포함한다.

- 실행한 exe 경로
- 실제로 확인한 기능 목록
- 실패해서 수정한 항목
- 아직 직접 확인하지 못한 항목과 이유
- 사용자가 바로 재현/확인할 수 있는 실행 방법
